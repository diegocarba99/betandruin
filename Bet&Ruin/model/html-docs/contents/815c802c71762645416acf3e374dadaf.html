Bet

UMLUseCase
Bet&Ruin :: Use Case View :: Bet

Description

Event flow

Main Flow

  1. User presses "Bet" button
  2. System get all the events the User has betted in, and procceses this information to create a Bet
  3. System computes the earnings for this bet per RP betted. Displayis this information to User
  4. User accepts to proceed with the bet
  5. System asks User for the amount of RP that will be betted
  6. User provides the quantity of RP he/she wants to bet and accepts this operation
  7. System gets the amount and procceses the bet assinging it to the User
  8. System informs User the bet has been succesfully proccesed

Alternative flow

A2. If no event has been selected, System informs user about the issue. Go to step 1.

B4. If user rejects bet, System informs User about the cancelation of the procces.

C6. If no correct amount of RP are provided by the User (too many RP, or no input at all), System informs User about the issue. Go to step 5.

Prequisites

User must have selected one or more bets in the browsing menu
