UMLUseCase
Bet&Ruin :: Use Case View :: Create event

Description

Event flow

Main Flow

  1. Admin selects a date
  2. System displays the events of this date
  3. Admin selects an event
  4. Admin introduces a question and minimum bet
  5. System adds new question with minimun bet to the selected event

Alternative flow

A2. There are no events on this date. Question cannot be added.

B5. If Admin doesn't provide a question or a valid number for minimum bet, then question cannot be added.

C5. If the event date is before current day, then question cannot be added.
